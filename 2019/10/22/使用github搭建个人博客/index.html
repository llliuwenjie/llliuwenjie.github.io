使用github/gitee + hexo搭建个人博客

发表于 2019-10-22 更新于 2021-01-19 分类于 博客

官方文档镇楼

https://hexo.io/zh-cn/docs/

node.js环境

git环境

博客地址

Github账户新建项目，项目要遵守格式：账户名.github.io，勾选Initialize this repository with a README
在建立好的项目右侧有settings按钮，点击它，向下拉到GitHub Pages，source选择master branch
等待一会会刷新，出现提示 Your site is published at https://xxx.github.io/
就是博客地址

hexo 文档

https://hexo.io/zh-cn/

安装Hexo

自己随意选中新建一个文件夹，通过命令安装Hexo

输入hexo -v，检查hexo是否安装成功

输入hexo init，初始化该文件夹，当看到Start blogging with Hexo则表明成功了。

输入npm install，安装所需组件

使用

配置

将Hexo与Github page联系起来，设置Git的user name和email

在安装blog的文件夹中鼠标右键点击Git Base Here。配置自己的用户名和邮箱

输入hexo s，开启服务器，访问该网址，正式体验Hexo。默认端口是4000，倘若提示端口被占，则使用命令hexo server -p 5000

配置免密登录

输入ssh-keygen -t rsa -C “邮箱”，连续回车即可生成秘钥，最后得到两个文件：id_rsa和id_rsa.pub(默认存储路径：c:\User\用户.ssh).

输入eval “$(ssh-agent -s)”，添加秘钥到ssh-agent

再输入ssh-add ~/.ssh/id_rsa，添加生成的ssh key到ssh-agent

添加秘钥到GitHub,换电脑后要重新执行

登录Github，点击头像下的settings，添加ssh and GPG keys

新建一个new ssh key，将id_rsa.pub文件里的内容复制上去。

在命令行输入ssh -T git@github.com，测试添加ssh是否成功。

配置Deployment，在其文件夹中，找到_config.yml文件，修改repository值。

新建一篇博客

在cmd命令：hexo new post “博客名”

建立成功后，在文件夹_posts目录下将会看到已经创建的文件,在生成并部署文章前，需要安装一个扩展：

使用编辑器编辑好文章后，使用命令：hexo g -d，生成以及部署

更改主题

在github上搜索：hexo-theme,上面有各种各样的主题，选择自己喜欢的主题git到安装目录的theme文件夹下。
也可以使用本地下载，再解压到theme
git clone 地址
再修改安装目录下的_config.yml,将其改为下载后的文件夹名

进行重新编译启动

本地测试

报错运行试试

实在不行，就把它删掉，然后重新生成和部署

就需要手动输入账号密码了
